<objective>
Introduce a config-driven cadence-policy registry and deterministic schedule computation before Phase 5 adds workers and notifications.

Purpose: The repo currently treats scheduling as a narrow `weekly` or `manual` branch on coverage entries. This plan upgrades that contract to named cadence policies such as weekly, biweekly, weekdays, monthly, and one constrained custom-weekdays option, while preserving the current coverage and run surfaces.
Output: Typed cadence-policy config, workspace-timezone schedule computation, additive coverage schedule fields, and backward-compatible CLI or API controls for selecting and inspecting schedule behavior.
</objective>
