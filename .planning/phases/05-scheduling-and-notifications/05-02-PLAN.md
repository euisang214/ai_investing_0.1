<objective>
Add the background refresh queue, worker infrastructure, review queue, and notification outbox while replacing the old universal post-gatekeeper pause rule.

Purpose: Phase 5 needs a true recurring-operations seam, but the repo must keep LangGraph as the reasoning runtime and preserve stable run inspection contracts. This plan introduces structured queued work and notifications, then routes all entrypoints through the same gatekeeper policy: pass or review auto-continues, fail stops into review plus immediate alerting, and provisional downstream analysis remains manual only.
Output: Structured queue and review persistence, worker services, a notification-event outbox plus delivery seam, additive operator controls, and a reconciled gatekeeper lifecycle that remains compatible with `show-run`, `continue-run`, `run-due-coverage`, `/coverage/run-due`, `/runs/{run_id}`, and `/runs/{run_id}/continue`.
</objective>
