# Phase 12-01: Cost Controls & CI/CD Validation

## Objective
Implement LLM usage tracking with configurable budget aborts (`AI_INVESTING_MAX_TOKENS_PER_RUN`) and introduce GitHub Actions CI/CD to enforce linting, type-checking, and tests.

## 1. Metric Tracking & Rate Cards
- Add `TokenUsage` struct in `domain/models.py` tracking `input_tokens`, `output_tokens`, `total_tokens`, and `estimated_cost_usd`.
- Modify `RunResultResponseData` in `api/main.py` and `RunRecord` runtime outputs to include `token_usage: TokenUsage | None`.
- Add `max_tokens_per_run: int | None` to `Settings` (env: `AI_INVESTING_MAX_TOKENS_PER_RUN`).
- Create `config/rate_cards.yaml` with USD cost values per token for models.
- Implement rate loading logic in a `RateCardRegistry`.
- Override `generate_structured_with_usage` in `providers/` (`openai_provider`, `anthropic_provider`, `gemini_provider`, `groq_provider`, `fake`, `openai_compatible_provider`, `resilient`) to use `include_raw=True` (or equivalent) for accurate token counts.

## 2. Runtime Accumulation & Budget Enforcement
- Add `track_usage(self, result: GenerationResult) -> None` to `RefreshRuntime`. It tracks token accumulations onto `self.run.metadata["token_usage"]` and applies rate cards to increment `estimated_cost_usd`.
- Add `is_budget_exceeded(self) -> bool` to `RefreshRuntime` that compares total tokens against the cap.
- Update all `provider.generate_structured(...)` LLM calls inside `RefreshRuntime` and `services.py` to use `provider.generate_structured_with_usage(...)` followed by `self.track_usage()`.
- Wrap `build_company_refresh_graph` nodes/edges inside `graphs/` logic: conditionally route to `END` or a terminal status when `is_budget_exceeded()` is True.
- Record `abort_reason: budget_exceeded` in the run metadata and reflect the stopped state gracefully instead of throwing unhandled exceptions.

## 3. CI/CD Pipeline Configuration
- Create `.github/workflows/ci.yml`.
- Run on `push` to `main` and `pull_request` on any branch.
- Add steps: Checkout, Install Python natively, `uv pip install -e ".[dev]"`, run `ruff check .`, `mypy --strict src tests`, and `pytest tests/`.
- Ensure step configures environment `AI_INVESTING_PROVIDER: fake`.

## Acceptance Criteria
- [ ] Requirements `COST-01`, `COST-03`, `CI-01`, `CI-02` are satisfied.
- [ ] API and CLI `RunResult` includes `token_usage` populated with inputs, outputs, and `estimated_cost_usd`.
- [ ] If `AI_INVESTING_MAX_TOKENS_PER_RUN` is hit, analysis aborts gracefully and reports reason in the returned result.
- [ ] `.github/workflows/ci.yml` is correctly defined for lint, type checks, and tests using `fake` provider without actual keys.
- [ ] All `tests/` pass under CI specifications with 100% test completion.
